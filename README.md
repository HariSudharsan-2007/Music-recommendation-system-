# EEG-Based Mood Classification and Recommendation System

## Overview
This project presents a Brain-Computer Interface (BCI) pipeline that classifies human emotional states from EEG signals and uses the predictions to drive environmental adaptations. The system utilizes machine learning to categorize user mood into three distinct states: `POSITIVE`, `NEUTRAL`, and `NEGATIVE`. The project features two practical integrations of the model's output: a mood-based music recommendation player and a dynamic graphical interface, effectively translating brain activity into personalized environmental feedback.

## Dataset
The project utilizes an EEG feature dataset (`emotions.csv`) containing 2,132 samples and 2,548 numerical features per sample, along with a final categorical target column. 

### Experimental Setup & Hardware
Data was collected from two subjects (1 male, 1 female) using a consumer-grade Muse EEG headband. The device recorded brain electrical activity via dry electrodes placed at four key positions:
* **TP9 & TP10** (Temporal-Parietal region): Linked to auditory and sensory processing.
* **AF7 & AF8** (Anterior-Frontal region): Associated with emotion regulation and decision-making.

Target emotions were evoked using specific video stimuli:
* **Positive**: Uplifting clips (e.g., the opening of *La La Land*, nature timelapses).
* **Negative**: Emotionally heavy scenes (e.g., scenes from *Up* and *Marley and Me*).
* **Neutral**: Six minutes of resting baseline brain activity.

### Data Structure
Rather than utilizing raw electrical waves, this dataset employs a statistical extraction strategy:
* **Features**: The dataset contains extensive statistical and frequency-domain features extracted from the raw EEG signals. Key feature categories include:
    * **Frequency Domain**: Fast Fourier Transform (`fft`) components.
    * **Statistical Moments**: Mean, standard deviation, variance, and higher-order moments.
    * **Covariance & Eigenvalues**: Covariance matrices (`covmat`), eigenvalues (`eigen`), and logarithmic matrix features (`logm`).
    * **Information Theory**: Shannon Entropy features (`entropy`).
    * **Correlation**: Cross-correlation features between the different channels (`correlate`).
    * **Extrema**: Minimum and maximum values within signal windows.
* **Target Labels**: The `label` column represents the categorized emotional state of the subject (`POSITIVE`, `NEUTRAL`, and `NEGATIVE`).

### Feature Importance
A Random Forest algorithm was used to rank feature importance via Mean Decrease in Impurity (Gini Importance). The model overwhelmingly identified that the frequency of the brainwaves—specifically high-variance Fast Fourier Transform (FFT) features—serves as the strongest indicator of emotional states, significantly outperforming raw amplitude metrics in cleanly separating the classes.

## Methodology and Code Implementation

### 1. Data Preprocessing
The initial phase of the implementation prepares the high-dimensional EEG data for the machine learning model:
* **Label Encoding**: The categorical target labels are transformed into numerical values (`POSITIVE` = 2, `NEUTRAL` = 1, `NEGATIVE` = 0) using `sklearn.preprocessing.LabelEncoder`.
* **Feature Standardization**: Given the varying scales of the extracted EEG features (e.g., high-variance FFT values vs. low-variance entropy scores), `StandardScaler` is applied to normalize the dataset, ensuring a mean of 0 and a standard deviation of 1 for all features.
* **Dimensionality Reduction**: To mitigate the curse of dimensionality and improve computational efficiency, Principal Component Analysis (PCA) is applied. The feature space is reduced from 2,548 variables to the top 100 principal components, retaining the most significant variance within the data.

### 2. Model Training and Evaluation
* **Data Splitting**: The dimensionally-reduced dataset is split into training (80%) and testing (20%) subsets.
* **Classification Algorithm**: A `RandomForestClassifier` (configured with 100 estimators and a fixed random state for reproducibility) is trained on the principal components.
* **Performance Metrics**: The model's performance is evaluated using accuracy scores, classification reports (precision, recall, f1-score), and a plotted confusion matrix. The implemented model achieves a highly robust classification accuracy of approximately 94.38% on the test set.

### 3. Downstream Applications
The predictions generated by the Random Forest model are routed into two localized Python scripts to demonstrate real-world applicability:
* **Music Recommendation Engine**: The system selects a random sample from the test set, predicts the emotional state, and maps the output to a designated local audio file. The `pygame.mixer` module is then utilized to load and play the corresponding track for a specified duration, providing an auditory feedback loop that matches the user's exact emotional state.
* **Dynamic Visual Environment**: A secondary script simulates a continuous BCI loop using the `pygame` display module. It routinely fetches the simulated "current mood" and dynamically updates the background color of a graphical window (e.g., Light Blue for positive, Gray for neutral, Dark Gray for negative). This serves as a foundation for integrating EEG predictions into video games or dynamic user interfaces.
